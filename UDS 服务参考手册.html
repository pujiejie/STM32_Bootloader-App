1. 会话控制 — SID 0x10

0x10Diagnostic Session ControlCAN+DoIP

切换 ECU 诊断会话模式。不同会话提供不同权限级别的服务访问。切换会话会复位安全解锁状态。

子功能

SF会话说明
0x01默认会话上电默认。仅 0x22 读数据、0x19 读 DTC
0x02编程会话⚠️ 收到后直接跳 Bootloader（复位重启）
0x03扩展会话允许 0x27 安全访问、0x2E 写数据、0x31 例程

报文示例

// 切换到扩展会话
请求: 10 03
响应: 50 03 00 32 13        // SF=03 P2=50ms P2*=5000ms

// 切换到编程会话 → ECU 立即复位进入 Bootloader
请求: 10 02
响应: 50 02 00 32 13        // 正响应发出后，ECU 在 20ms 内复位

对应代码

UDS_Service_10_App() — 第 88 行 uds.c

2. ECU 复位 — SID 0x11

0x11ECU ResetCAN+DoIP

强制 ECU 硬件复位。正响应发出后 20ms 内执行 NVIC_SystemReset()。

子功能

SF复位类型
0x01硬复位 (Hard Reset)
0x03软复位 (等同于硬复位)

报文示例

请求: 11 01
响应: 51 01                 // ECU 随后复位

3. 清除诊断信息 — SID 0x14

0x14Clear Diagnostic InformationCAN+DoIP

清除指定的 DTC 故障码。

参数

参数含义
0xFFFFFF清除 全部 DTC
指定 DTC Code（3字节）仅清除匹配的单个 DTC

报文示例

// 清除全部 DTC
请求: 14 FF FF FF
响应: 54

// 清除指定 DTC
请求: 14 00 F1 00           // 只清除 0xF100
响应: 54

4. 读 DTC — SID 0x19

0x19Read DTC InformationCAN+DoIP

读取故障码信息。支持按状态掩码过滤。需 DTC 功能开启（0x85 服务控制）。

子功能

SF功能参数
0x01按状态掩码统计数量[19 01 mask]
0x02按状态掩码列出 DTC[19 02 mask]
0x06列出全部 DTC（忽略掩码）[19 06]

状态掩码 (bit flags)

位值名称含义
bit 00x01Test Failed最近一次检测失败
bit 30x08Confirmed DTC故障已确认，需诊断仪清除
bit 50x20Pending DTC疑似故障，等待确认
bit 70x40Warning Indicator仪表盘故障灯点亮

掩码可组合：0x09 = Test Failed + Confirmed；0xFF = 全部。

响应格式

SF=0x01 返回：59 01 00 00 [cnt]

SF=0x02/0x06 返回：59 02 00 00 [DTC_3B+Status_1B]*N

报文示例

// 统计已确认的 DTC 数量
请求: 19 01 08
响应: 59 01 00 00 03        // 3 个已确认

// 列出全部 DTC
请求: 19 02 FF
响应: 59 02 00 00
          00F100 49              // 已确认+警告灯+测试失败
          00F101 08              // 仅已确认, 已消失
          00F102 20              // 待确认, 活跃中
          00F103 08              // 仅已确认, 已消失

// 只查 Pending 的 DTC
请求: 19 02 20
响应: 59 02 00 00
          00F102 20              // 只有这一个是 pending

// 查全部（不用掩码）
请求: 19 06
响应: 59 06 00 00
          00F100 49
          00F101 08
          00F102 20
          00F103 08

DID数据字节数示例值

F1 00总电压 (mV)212600 mV
F1 01电流 (mA, 有符号)2-2300 mA
F1 02SOC (%)178%

F1 04单体最高电压 (mV)23650 mV
F1 05单体最低电压 (mV)23580 mV

报文示例

// 读电流（负数 = 放电）
请求: 22 F1 01
响应: 62 F1 01 F7 04          // 0xF704 = -2300 mA = -2.3A (放电)

6. 安全访问 — SID 0x27

0x27Security Access需扩展会话

挑战-应答安全认证。解锁后才能执行 0x2E 写数据和 0x31 例程控制。

子功能

SF步骤
0x01请求种子 (Request Seed)
0x02发送密钥 (Send Key)

锁定机制：连续 3 次密钥错误后锁定 10 秒，期间返回 NRC 0x37。

密钥算法

Key = ((~Seed ^ 0x5A5A5A5A) + 0x3618BC91) & 0xFFFFFFFF

报文示例

// 步骤 1: 请求种子
请求: 27 01
响应: 67 01 A5 C3 E1 F7    // Seed = 0xA5C3E1F7

// 步骤 2: 计算并发送密钥
// Key = ((~0xA5C3E1F7 ^ 0x5A5A5A5A) + 0x3618BC91)
//     = ((0x5A3C1E08 ^ 0x5A5A5A5A) + 0x3618BC91)
//     = (0x00024452 + 0x3618BC91)
//     = 0x361B00E3
请求: 27 02 36 1B 00 E3
响应: 67 02                 // 解锁成功！现在可以写数据了

7. 通信控制 — SID 0x28

0x28Communication ControlCAN+DoIP

控制 ECU 通信状态，可让 ECU 进入静默模式停止响应。

报文示例

// 开启静默模式 (不响应)
请求: 28 00 03
响应: 68 00 03

// 恢复正常通信
请求: 28 00 00
响应: 68 00 00

8. 写数据 — SID 0x2E

0x2EWrite Data By Identifier需 0x27 解锁

通过 DID 修改 BMS 配置参数。必须先通过 0x27 安全访问解锁。

可写的 DID

DID数据格式
F1 90VIN 码17 字节 ASCII
F1 89软件版本号变长 ASCII
F1 02SOC1 字节 (0-100)
F1 08充放电状态1 字节
F1 01电流2 字节有符号 (mA)
F1 80充电使能1 字节 (1/0)
F1 81均衡使能1 字节 (1/0)

报文示例

// 修改 SOC 为 100%
请求: 2E F1 02 64
响应: 6E F1 02

// 修改 VIN
请求: 2E F1 90 4E 45 57 56 49 4E 30 30 30 30 30 30 30 30 30 30 30
响应: 6E F1 90              // VIN = "NEWVIN00000000000"

9. 例程控制 — SID 0x31

0x31Routine Control需 0x27 解锁

执行预定义的诊断例行程序。必须先通过 0x27 安全访问解锁。

仅支持 SF=0x01（启动例程），不支持 0x02（停止）和 0x03（查询结果）。

支持的例程 ID (RID)

RID例程响应
E0 01BMS 自检71 E0 01 01 (status=pass)
E0 02复位 SOC 到 100%71 E0 02 01
E0 03复位 SOH 到 100%71 E0 03 01

报文示例

// 执行 BMS 自检
请求: 31 01 E0 01
响应: 71 E0 01 01           // 自检通过

// 复位 SOC 到 100%
请求: 31 01 E0 02
响应: 71 E0 02 01           // SOC 已设为 100

10. 测试仪在线 — SID 0x3E

0x3ETester PresentCAN+DoIP

心跳信号，防止 ECU 自动退出当前会话。不需要响应时设 bit7=1。

报文示例

// 需要响应
请求: 3E 00
响应: 7E

// 不需要响应 (抑制正响应)
请求: 3E 80
// 无响应

11. DTC 设置控制 — SID 0x85

0x85Control DTC SettingCAN+DoIP

开关 DTC 记录功能。关闭后 0x19 读 DTC 会返回 NRC 0x22。

子功能

SF功能
0x01启用 DTC 记录 (On)
0x02禁用 DTC 记录 (Off)

报文示例

// 禁用 DTC
请求: 85 02
响应: C5 02

// 此时读 DTC 会失败
请求: 19 01 FF
响应: 7F 19 22              // NRC 0x22 = Conditions Not Correct

// 重新启用
请求: 85 01
响应: C5 01

12. 固件刷写 — SID 0x34 / 0x36 / 0x37

0x34/36/37Request Download / Transfer Data / Exit Transfer仅 Bootloader (CAN)

固件刷写三件套。仅支持 CAN 通道，APP 收到会返回 NRC 0x22。
需先通过 0x10 02 编程会话 + 安全解锁 + 0x31 FF00 擦除 Flash。

报文示例

// 0x34: 请求下载
// 格式: 34 [DataFmtID] [AddrSizeFmt] [Addr 4B BigEndian] [Size 4B BigEndian]
请求: 34 00 44 08 01 00 00 00 0F 00 00
       SID DF=00(无加密) AL=44(4+4)    Addr=0x08010000   Size=0x000F0000(960KB)
响应: 74 40 80 00             // MaxBlockSize=0x8000=32KB/块

// 前置条件: 编程会话(0x10 02) + 安全解锁(0x27) + Flash已擦除(0x31 FF00)
// 地址限制: Addr 必须 >= 0x08010000 (保护 Bootloader 区域)


完整刷写流程


// 步骤 1: 编程会话 → 跳到 Bootloader (CAN 通信)
请求: 10 02
响应: 50 02 00 32 13          // 随后 ECU 复位进入 Bootloader

// 步骤 2: 安全访问 (Seed/Key)
请求: 27 01
响应: 67 01 xx xx xx xx       // 获取 Seed
请求: 27 02 xx xx xx xx       // 发送 Key
响应: 67 02

// 步骤 3: 擦除 APP 区域
请求: 31 01 FF 00
响应: 71 FF 00 01             // 擦除成功 (擦除后签名变 0xFFFFFFFF)

// 步骤 4: 请求下载
请求: 34 00 44 08 01 00 00 00 0F 00 00
响应: 74 40 80 00             // MaxBlockSize=32KB

// 步骤 5: 分块传输
// 每块 BlockSeq 递增，固件数据 32KB/块
请求: 36 01 [32KB]  → 76 01
请求: 36 02 [32KB]  → 76 02
// ... 重复直到全部发完 ...

// 步骤 6: 退出传输 + CRC32 校验
请求: 37 [CRC32 4B]
响应: 77                      // CRC 通过, 写入签名 0xB00710AD

// 步骤 7: ECU 复位
请求: 11 01
响应: 51 01                      // ECU 复位, 启动新 APP



// 0x36: 传输数据
// 格式: 36 [BlockSeq 1B] [Data N Bytes]
// BlockSeq 从 1 开始递增，必须连续，Bootloader 会校验序号
请求: 36 01 [32KB 固件数据]    // BlockSeq=01
响应: 76 01                   // 确认收到块01

请求: 36 02 [32KB 固件数据]    // BlockSeq=02
响应: 76 02

// ... 重复直到全部数据发完 ...

// 0x37: 退出传输 (带 CRC32 校验)
// 格式: 37 [CRC32 4B BigEndian] (CRC32 可选，>=5字节时校验)
请求: 37 A1 B2 C3 D4          // 可选 CRC32 校验值
响应: 77                   // CRC32 校验通过, 写入签名 0xB00710AD @0x080FFFF8

CAN 帧格式 (ISO-TP)

帧类型PCI Byte含义数据长度
单帧 (SF)0x0NN = 数据长度 (0-7)1 帧 ≤ 7 字节
首帧 (FF)0x1N NNNNN = 数据总长度 (8-4095)第1帧含 6 字节数据
连续帧 (CF)0x2SS = 序号 1..15, 循环每帧含 7 字节数据
流控帧 (FC)0x3 0 BS STBS=连续帧数 ST=最小间隔ECU 响应
// 单帧示例: 发送 "10 03" (2 字节)
CAN ID: 7E0  Data: 02 10 03 00 00 00 00 00  // PCI=0x02 表示长度2

// 多帧示例: 发送 17 字节 VIN (首帧 + 2 连续帧)
CAN ID: 7E0  Data: 10 11 22 F1 90 FC X2 0  // FF: PCI=0x10, 总长=0x011=17B
CAN ID: 7E8  Data: 30 00 00 00 00 00 00 00  // FC: BS=0(不限), ST=0(最快)
CAN ID: 7E0  Data: 21 26 B M S 7 8 9 0     // CF: PCI=0x21 序号1
CAN ID: 7E0  Data: 22 1 2 3 00 00 00 00      // CF: PCI=0x22 序号2

DoIP 帧格式

// 所有 DoIP 消息前 8 字节为通用头:
 Byte 0:    Version (0x02)
 Byte 1:    Version Inverse (~0x02 = 0xFD)
 Byte 2-3:  Payload Type (大端)
 Byte 4-7:  Payload Length (大端)

// 诊断消息 (0x8001) 的 Payload 格式:
 Byte 0-1:  Source Address (测试仪: 0x0E00)
 Byte 2-3:  Target Address (ECU: 0x0E80)
 Byte 4+:   UDS 数据

// 发送 "22 F1 02" (读 SOC) 的完整 DoIP 帧:
TCP: 02 FD 80 01 00 00 00 07  // Header: version=02 inv=FD type=0x8001 len=7
               0E 00 0E 80        // SA=0x0E00 TA=0x0E80
               22 F1 02           // UDS: Read DID 0xF102 (SOC)

// ECU 响应:
TCP: 02 FD 80 01 00 00 00 06  // Header: type=0x8001 len=6
               0E 80 0E 00        // SA=0x0E80 TA=0x0E00 (互换!)
               62 F1 02 4E     // UDS: 正响应 DID F102 = 0x4E (78%)
